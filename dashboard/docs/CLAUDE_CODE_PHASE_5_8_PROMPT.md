## 2. Step-by-step

### Step 1 — Update the Experiments list API endpoint

In `dashboard/app/api/experiments/route.ts`, the existing GET handler returns the list of experiments. Extend it to also return two aggregate counts in the response.

Locate the GET handler. Before returning the response, add:

```typescript
// Count SRM alerts across running experiments
let srmAlerts = 0;
for (const exp of experiments) {
  if (exp.status !== "running") continue;
  try {
    const { computeExperimentResults } = await import("@/lib/experiment-results");
    const results = await computeExperimentResults(exp.id, customer.id);
    if (results?.srm_flag) srmAlerts += 1;
  } catch (err) {
    console.warn(`Failed to load results for SRM check on ${exp.id}:`, err);
  }
}

// Count readouts generated
const readoutsRow = await queryOne<{ count: string }>(
  `SELECT COUNT(*)::text as count
   FROM readouts r
   JOIN experiments e ON r.experiment_id = e.id
   WHERE e.customer_id = $1`,
  [customer.id]
);
const readoutsGenerated = parseInt(readoutsRow?.count || "0", 10);
```

Then add these two fields to the response object that the endpoint returns:

```typescript
return corsResponse({
  experiments: experimentsList,           // existing
  // ... any other existing fields ...
  srm_alerts: srmAlerts,                  // NEW
  readouts_generated: readoutsGenerated,  // NEW
});
```

If the existing handler already imports `queryOne` and `computeExperimentResults`, no new imports needed. If not, add them at the top of the file.

## 5. Notes for the implementer

**Why no sparklines here**: The Dashboard is workspace-overview territory — sparklines provide useful trend context. The Experiments page is an inventory view focused on the experiment cards below the KPI strip. Adding sparklines to the Experiments KPI strip would compete visually with the experiment cards. Clean numbers keep the KPI strip as summary context, with attention flowing down to the experiment cards.

**About icon color when SRM = 0**: When all randomization is healthy, the AlertTriangle icon should appear in neutral gray (`text-[#727785]`), not green. Green would suggest "positive metric" but AlertTriangle is fundamentally a warning shape — gray reads as "no alerts" cleanly.

**About variable name `data`**: The example code above assumes the page uses a variable named `data` for the API response. Inspect the existing page; the actual variable name (likely `experiments`, `apiData`, `response`, etc.) should match what's used for the Win Rate / Est. Impact cards currently. Replace `data.srm_alerts` with `<actual-var>.srm_alerts` etc.
